# Spec: Initial Implementation

## Goal

Ship a minimal but usable AHP terminal client.

## Must-have

1. Connect to server (`ws://`/`wss://`)
2. List sessions
3. Create session
4. Open session and send messages
5. Display streaming responses
6. Support tool confirmation prompts

## Nice-to-have

- Dev Tunnel connection flow
- Folder picker for working directory
- Better scrolling and viewport behavior

## Out of scope (initial)

- Plugin system
- Rich markdown rendering
- Multi-window terminal layouts
